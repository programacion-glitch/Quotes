# H2O Quote RPA - Arquitectura Técnica

## 🏗️ Diseño de Arquitectura

### Principios de Diseño

1. **Modularidad**: Cada módulo tiene una responsabilidad única y bien definida
2. **Extensibilidad**: Fácil agregar nuevos tipos de documentos o flujos
3. **Mantenibilidad**: Código limpio, documentado y con convenciones claras
4. **Robustez**: Manejo de errores y logging en cada capa

### Separación de Responsabilidades

```
┌─────────────────────────────────────────────────┐
│                  main.py                        │
│            (Orquestador Principal)              │
└─────────────────────────────────────────────────┘

        ┌─────────────┼─────────────┬─────────────┐
        │             │             │             │
   ┌────▼────┐  ┌────▼────┐  ┌────▼────┐  ┌────▼────┐
```

### 2. `commodity_matcher.py`

**Responsabilidad**: Identificar el tipo de negocio mediante fuzzy matching del campo commodity

**Entrada**:
- String del commodity extraído (ej: "DIRT, SAND & GRAVEL 100%")
- Lista de tipos de negocio del Excel

**Salida**:
- Tipo de negocio matched (ej: "DIRT, SAND & GRAVEL")
- Score de similitud (0-100)

**Lógica de Matching**:
1. Normalización del texto:
   - Remover porcentajes (%XX)
   - Convertir a mayúsculas
   - Remover espacios extras
   - Remover caracteres especiales innecesarios

2. Comparación fuzzy:
   - Usar `difflib.SequenceMatcher` o `fuzzywuzzy`
   - Calcular score de similitud para cada tipo
   - Threshold mínimo: 60%

3. Retornar el match con mayor score

**Ejemplo**:
```python
Input:  "SAND & GRAVEL 100%"
Types:  ["DIRT, SAND & GRAVEL", "FRAC SAND", "GRAIN"]
Output: ("DIRT, SAND & GRAVEL", 78.5)
```

**Funciones principales**:
```python

    """Normaliza el string del commodity"""

def find_best_match(
    commodity: str, 
    business_types: list[str],

) -> tuple[str, float] | None:
    """
    Encuentra el mejor match usando fuzzy matching.
    
    Returns:
        (matched_type, score) o None si no hay match > threshold
    """
```

### 3. `excel_config.py`

**Responsabilidad**: Leer configuración desde el Excel estandarizado

**Entrada**:
- Ruta al archivo Excel

**Salida**:
- Configuración completa: tipos de negocio, mensajes, reglas

**Estructura del Excel**:
```
Columna A: TIPO DE NEGOCIO
Columna B: COMENTARIOS (mensaje a enviar)
Columna C: ... (campos adicionales)
```

**Tipos de Negocio (Ejemplos)**:
- AGRICULTURAL
- AUTO HAULER
- CARGO VAN
- DIRT, SAND & GRAVEL
- DRY VAN / REEFER
- FLATBED
- HAZARDOUS MATERIALS
- ... (ver lista completa en Excel)

**Funciones principales**:
```python
def load_business_types(excel_path: str) -> list[str]:
    """Carga todos los tipos de negocio del Excel"""

def get_message_for_type(
    excel_path: str, 
    business_type: str
) -> str:
    """Obtiene el mensaje correspondiente a un tipo de negocio"""

def load_config(excel_path: str) -> dict:
    """Carga toda la configuración del Excel en un dict"""
```

**Estado**: 🏗️ En desarrollo

### 4. `message_builder.py`

**Responsabilidad**: Construir mensaje personalizado según tipo de negocio

**Entrada**:
- Datos extraídos del PDF (dict)
- Tipo de negocio identificado
- Plantilla de mensaje del Excel

**Salida**:
- Mensaje formateado listo para enviar

**Lógica**:
1. Obtener plantilla del Excel según tipo
2. Reemplazar placeholders con datos del PDF
3. Aplicar formateo personalizado
4. Validar que el mensaje esté completo

**Placeholders soportados** (ejemplos):
- `{applicant_name}`
- `{business_name}`
- `{commodity}`
- `{liability_limit}`

**Funciones principales**:
```python
def build_message(
    extracted_data: dict,
    business_type: str,
    template: str
) -> str:
    """Construye el mensaje final reemplazando placeholders"""

def validate_message(message: str) -> bool:
    """Valida que no queden placeholders sin reemplazar"""
```

## 🔄 Flujo de Datos Completo

```
1. main.py recibe ruta al PDF
   ↓
2. pdf_extractor.extract_quote_data(pdf_path)
   → Returns: extracted_data (dict)
   ↓
3. excel_config.load_config(excel_path)
   → Returns: config (business_types, messages)
   ↓
4. commodity_matcher.find_best_match(
      extracted_data["commodities"],
      config["business_types"]
   )
   → Returns: (matched_type, score)
   ↓
5. excel_config.get_message_for_type(matched_type)
   → Returns: template (str)
   ↓
6. message_builder.build_message(
      extracted_data,
      matched_type,
      template
   )
   → Returns: final_message (str)
   ↓
7. [FUTURO] email_sender.send(final_message)
```
