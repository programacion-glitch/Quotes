# H2O Quote RPA - Automated Quote Processing

## 📋 Overview

Sistema RPA (Robotic Process Automation) para automatizar el procesamiento de cotizaciones de seguros comerciales (Blue Quote PDFs). El sistema extrae datos de PDFs, clasifica el tipo de negocio basado en commodities, y genera respuestas automatizadas.

## 🎯 Objetivo

Automatizar el flujo completo desde la recepción de un email con PDF de cotización hasta el envío de respuesta personalizada según el tipo de negocio del cliente.

## 📁 Estructura del Proyecto

```
H2O_Quote_RPA/
├── config/
│   └── CHECK LIST (2)_ESTANDARIZADO.xlsx  # Configuración de tipos de negocio y mensajes
├── data/
│   ├── input/          # PDFs entrantes (a procesar)
│   └── output/         # JSONs extraídos y logs
├── modules/
│   ├── __init__.py
│   ├── pdf_extractor.py      # Extracción de datos de PDFs (Blue Quote)
│   ├── commodity_matcher.py  # Fuzzy matching: commodity → tipo de negocio
│   ├── excel_config.py       # Lectura de configuración desde Excel
│   └── message_builder.py    # Construcción de mensajes según tipo
├── BlueQuote/
│   ├── extract_quote.py      # Script base de extracción (core)
│   └── *.pdf                 # PDFs de ejemplo/prueba
├── main.py                   # Orquestador principal del flujo
├── requirements.txt          # Dependencias Python
├── README.md                # Este archivo
└── ARCHITECTURE.md          # Documentación técnica detallada
```

## 🚀 Flujo de Procesamiento

```
1. Email recibido con PDF
     ↓
2. PDF extraído → data/input/
     ↓
3. pdf_extractor.py → Extrae datos (JSON)
     ↓
4. commodity_matcher.py → Identifica tipo de negocio (fuzzy matching)
     ↓
5. message_builder.py → Genera mensaje personalizado
     ↓
6. Envío de respuesta por email
     ↓
7. Archivado → data/output/
```

## 📦 Módulos Principales

### ✅ Implementados

- **`BlueQuote/extract_quote.py`**: Extractor PDF → JSON
  - Extrae: Applicant Info, Drivers, Vehicles, Coverages
  - Maneja checkboxes, text fields, y campos dinámicos

### 🏗️ En Desarrollo

- **`commodity_matcher.py`**: Clasificador fuzzy de commodities
- **`excel_config.py`**: Lector de configuración
- **`message_builder.py`**: Constructor de mensajes

### 📋 Planeados

- Email receiver (IMAP/Exchange)
- Email sender (SMTP/Graph API)
- Logger centralizado
- Error handler y retry logic

## 🛠️ Tecnologías

- **Python 3.x**
- **pdfplumber**: Extracción de PDFs
- **openpyxl**: Lectura de Excel
- **difflib/fuzzywuzzy**: Fuzzy matching
- (Futuro) **Exchange/SMTP**: Email automation

## 📝 Convenciones de Código

- **Modularidad**: Un módulo = Una responsabilidad
- **Nombres descriptivos**: `commodity_matcher.py` no `utils.py`
- **Funciones pequeñas**: Max 50 líneas por función
- **Type hints**: Siempre que sea posible
